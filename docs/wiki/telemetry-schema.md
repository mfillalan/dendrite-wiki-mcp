# Telemetry Ingestion Schema

This page defines the first stable Supabase schema for the current telemetry uploader. The goal is a direct, inspectable insert path with minimal transformation before any richer hosted analytics layer exists.

## Current Target

- The uploader posts directly to `/rest/v1/benchmark_events`.
- The default table name is `benchmark_events`, overridable with `DENDRITE_WIKI_TELEMETRY_SUPABASE_TABLE`.
- Each upload sends one `telemetry_summary` row.

## Column Contract

| Column | Type | Notes |
|---|---|---|
| `id` | `bigint identity` | Server-side receipt id. |
| `schemaVersion` | `integer` | Fixed to `1` for this contract. |
| `installationId` | `uuid` | Random local installation id. |
| `projectId` | `uuid` | Random local project id. |
| `packageVersion` | `text` | Current package version or `null`. |
| `event` | `text` | Fixed to `telemetry_summary` in this milestone. |
| `timestamp` | `timestamptz` | Client-side event timestamp for the upload row. |
| `sharingMode` | `text` | Fixed to `opt-in`. |
| `clientProfiles` | `jsonb` | JSON array of client profile labels. |
| `metrics` | `jsonb` | JSON object with aggregate counters only. |
| `insertedAt` | `timestamptz` | Server-side receipt timestamp. |

The table intentionally keeps the uploader's exact camelCase property names. That keeps the direct PostgREST insert auditable and avoids a hidden rename layer before the product has real telemetry volume.

## SQL Artifact

The current SQL artifact is published in the docs public assets at `/dendrite-telemetry-supabase-schema.sql`.

```sql
create table if not exists public.benchmark_events (
  id bigint generated by default as identity primary key,
  "schemaVersion" integer not null check ("schemaVersion" = 1),
  "installationId" uuid not null,
  "projectId" uuid not null,
  "packageVersion" text,
  event text not null check (event = 'telemetry_summary'),
  "timestamp" timestamptz not null,
  "sharingMode" text not null check ("sharingMode" = 'opt-in'),
  "clientProfiles" jsonb not null default '[]'::jsonb check (jsonb_typeof("clientProfiles") = 'array'),
  metrics jsonb not null check (jsonb_typeof(metrics) = 'object'),
  "insertedAt" timestamptz not null default timezone('utc', now())
);

create index if not exists benchmark_events_timestamp_idx
  on public.benchmark_events ("timestamp" desc);

create index if not exists benchmark_events_installation_project_idx
  on public.benchmark_events ("installationId", "projectId");

create index if not exists benchmark_events_metrics_gin_idx
  on public.benchmark_events
  using gin (metrics jsonb_path_ops);
```

## Access Notes

- This milestone assumes the key in `DENDRITE_WIKI_TELEMETRY_SUPABASE_KEY` is supplied from a local shell or secrets manager, not browser code.
- Keep the access model narrow. If you later move to an Edge Function or another ingestion layer, preserve this wire contract or version it deliberately.
- The current schema is optimized for auditability and low ceremony, not for every future analytics query.

## Future Evolution

- Add richer tables only after the current payload stays stable across real opted-in projects.
- Prefer additive versioned changes instead of rewriting historical rows.
- If a later schema wants snake_case or normalization, add a deliberate translation layer rather than silently changing the payload.

See [Privacy And Telemetry Disclosure](./privacy-telemetry-disclosure.md) for the human-readable explanation of what the payload means and what never leaves the machine.
